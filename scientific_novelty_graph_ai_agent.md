# Научная новизна: графовый подход в работе ИИ-агента

## 1. Введение

Интеграция графов знаний (Knowledge Graphs, KG) в архитектуры ИИ-агентов для телемедицины представляет собой актульное преспективное направление исследований. 
Python-файл (`telemed_conf_ai_assistant.ipynb`) реализует телемедицинского ИИ-ассистента, который использует граф медицинских знаний (KG) для усиления поддержки клинических решений через многошаговое рассуждение, верификацию утверждений и генерацию рекомендаций на основе доказательной базы.

## Постановка проблемы
Традиционные LLM полагаются на закодированные в весах искусственных нейронных сетей знания, что делает их склонными к генерации правдоподобных, но неверифицированных медицинских утверждений. Ограничивая рассуждения агента заданным промтом. Исследования демонстрируют, что RAG с применением графов достигает **95% точности с нулевым уровнем галлюцинаций** по сравнению с **34% у обычных LLM** при синтезе медицинских доказательств 

> [paper link](https://www.nature.com/articles/s41586-023-06291-2)
> Singhal, K., Azizi, S., Tu, T. et al. Large language models encode clinical knowledge. Nature 620, 172–180 (2023)

## Что нового в науке

В отличие от стандартного RAG, который извлекает изолированные текстовые фрагменты, агенты на основе графов знаний выполняют **траверсное рассуждение** (traversal reasoning) по сущностям и отношениям. Для этого сделана база знаний `CLINICAL_REQUIREMENTS_DB` и база сущностей и отношений `MEDICAL_KNOWLEDGE_GRAPH`. Объединение этих баз в граф медицинских знаний через функцию `query_clinical_knowledge_graph_ru`, которая находит не только прямые совпадения, но и соседей в графе в цепочке рассуждений (например, «гипертензия» → «медикаменты» → «алгоритм действия врача» → «степень уверенности»). 

Подобный подход работы с графами в виде структур для восходящего/нисходящего рассуждения рассмотрен в работе 
[paper link](https://dl.acm.org/doi/10.1145/3696410.3714782)

> Xuejiao Zhao, Siyan Liu, Su-Yin Yang, and Chunyan Miao. 2025. MedRAG: Enhancing Retrieval-augmented Generation with Knowledge Graph-Elicited Reasoning for Healthcare Copilot. In Proceedings of the ACM on Web Conference 2025 (WWW '25). Association for Computing Machinery, New York, NY, USA, 4442–4457.

В работе также используется графовый подход [paper link](https://www.mdpi.com/1999-5903/17/10/471)
> Mushtaq, S.; Trovati, M.; Bessis, N. Explainable AI-Based Semantic Retrieval from an Expert-Curated Oncology Knowledge Graph for Clinical Decision Support. Future Internet 2025, 17, 471. https://doi.org/10.3390/fi17100471

Новизна предложенного в работе подхода - использование клинических рекомендаций `CLINICAL_REQUIREMENTS_DB` (с точки зрения науки - вместо одно - применяются две базы, объединенные в граф). В коде определён граф знаний с тремя типами узлов, каждый из которых несёт уникальные признаки, отсутствующие в классических онтологиях: 
* Узлы типа claim (утверждения) - это данные из медицинской практики
* Узлы типа guideline (клинические рекомендации) - это данные из клинических рекомендаций
* Узлы типа entity (медицинские сущности) - это виды медицинских услуг
Эти узлы связаны рёбрами, вес рёбер показывает степень уверенности в зависимостях между узлами (уровень доказательности из клинических рекомендаций). В работе реализована функция `TelemedicineAnalysisAgent` с разделением  **детерминированных знаний (KG) и  рассуждения (LLM-агент Mistal)**, что обеспечивает итеративное уточнение без галлюцинаций.

Алгоритм `generate_operator_tips_from_graph` извлекает признаки и формирует правила рассуждений. Узлы `guideline` содержат атрибуты `url` и `last_updated`, обеспечивая **атрибуцию источников**. 


**Итого** - разработан новый алгоритм, создающий пространство признаков для клинических задач. Стандартные агенты могут использовать графовые структуры для формирования правил, но никто не применял взвешенный граф с подобным пространством признаков из клинических рекомендаций. 


## Сравнительная таблица: агент с графом и без графа

| Характеристика | Базовый агент (без графа) | Агент с KG |
|----------------|---------------------------|------------|
| **Ретривал** | Ключевое/нечеткое совпадение по текстовой БД | Структурированный траверс графа |
| **Контроль галлюцинаций** | Отсутствует (полагается на параметрические знания LLM) | Нулевой (ограничен триплетами KG) |
| **Обнаружение дезинформации** | Не способен выявить ложные утверждения пациента | Активное опровержение через метаданные вердиктов |
| **Объяснимость** | Непрозрачный (текст на входе — текст на выходе) | Прозрачные пути рассуждения через траверс ребер |
| **Многошаговый вывод** | Одношаговый ретривал | Многошаговая цепочка сущность–отношение |
| **Происхождение источников** | Статические ссылки БД | Динамическая атрибуция URL + временная метка |
| **Клиническая полезность** | Общие рекомендации | Контекстуальные советы оператору с оценками достоверности |

##  Выводы
Код демонстрирует ключевые архитектурные принципы, валидированные современными исследованиями SOTA-уровня: графы знаний служат **верифицируемыми фактологическими якорями**, ограничивающими генерацию LLM, а агенты делают сложные структурированные знания доступными через естественный язык.
